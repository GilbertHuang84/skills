### session-log.md — 对话摘要

```markdown
# 对话日志

## Session {编号} — {日期}
- 当前阶段：[Phase 0 / 第一层 / 第二层 / 第三层]
- 本轮完成：[做了什么]
- 关键发现：[发现了什么]
- 下次计划：[下一步做什么]
- 存档更新：[更新了哪些文件]
```

## 存档操作规范

1. **每次对话结束前必须存档**
2. **capability-plan.md 是核心文件**，记录补强内容和进度
3. **environment-map.md 和 direction-anchors.md** 在对应层级完成后创建
4. **session-log.md 按时间倒序排列**
5. **存档内容使用中文**（与用户对话语言一致）
6. **所有分析必须基于具体证据**，避免主观臆断

## 存档路径

| 文件 | 路径 |
|------|------|
| capability-plan.md | `data/{用户标识}/capability/capability-plan.md` |
| environment-map.md | `data/{用户标识}/capability/environment-map.md` |
| direction-anchors.md | `data/{用户标识}/capability/direction-anchors.md` |
| session-log.md | `data/{用户标识}/session-logs/capability-builder.md` |
